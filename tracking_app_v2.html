How To Use

Setup guide, daily habits, Firebase configuration, Notion journal setup, and a reference for every tab — all in one place.

🔥 Step 1 — Firebase Setup (10 minutes, do this first)
One-Time Setup — Required Before First Sign-In

Create Your Firebase Project

Firebase is Google's free database service. You create your own project so your data belongs entirely to you. The free tier is more than sufficient for this app forever.

STEP 1 — CREATE PROJECT
Go to console.firebase.google.com → Click Add project → Name it Scratch Project Tracker → Disable Google Analytics → Click Create.
STEP 2 — CREATE FIRESTORE DATABASE
In your project → Left menu → Firestore Database → Create database → Start in production mode → Select region: europe-west2 (London) → Enable.
STEP 3 — SET SECURITY RULES
In Firestore → Rules tab → Replace all text with:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null
      && request.auth.uid == userId;
    }
  }
}
→ Click Publish.
STEP 4 — ENABLE GOOGLE SIGN-IN
Left menu → Authentication → Get started → Sign-in method → Google → Enable → Add your email as support email → Save.
STEP 5 — ADD AUTHORISED DOMAIN
Authentication → Settings → Authorised domains → Add domain → enter: asbremner-ai.github.io → Add. This allows sign-in from your GitHub Pages site.
STEP 6 — GET YOUR CONFIG & PASTE INTO THE APP
Project Overview → </> web icon → Register app (name: Scratch Tracker) → Copy the firebaseConfig object → Open tracking_app.html in a text editor → Find the comment PASTE YOUR FIREBASE CONFIG HERE → Replace the placeholder values with your real config → Save → Upload to GitHub.
◻ Step 2 — Notion Journal Setup (15 minutes)
Set Up Your Notion Journal Workspace

Build the Guide 17 Journal Structure in Notion

Notion's free personal plan is sufficient. The journal template below mirrors the Guide 17 Progress Journal structure exactly — weekly reviews, mental game ratings, and quarterly checkpoints.

STEP 1
Sign up or log in at notion.so. Create a new page. Title it: "The Scratch Project — Journal".
STEP 2 — WEEKLY REVIEW DATABASE
Type /database → Full page database → Name: Weekly Reviews. Add these properties:

• Date (type: Date)
• SG Total (Number, format: Number)
• SG OTT / APP / ARG / PUTT (4× Number properties)
• HRV Trend (Select: Improving / Stable / Declining)
• Practice Structure (Select: Mostly Blocked / Mixed / Mostly Random)
• Priority (Text)
• Best Moment (Text)
• Biggest Challenge (Text)
• Key Learning (Text)
• Competitions (Text)
• Decision Quality avg (Number 1–10)
• Emotional Control avg (Number 1–10)
• Status (Select: Draft / Complete)
STEP 3 — QUARTERLY REVIEW SECTION
Below the Weekly database, create a new page: Quarterly Reviews. Create 8 sub-pages (Q1 Sep 2026 through Q8 Jun 2028). Each page uses the 14-question template from Guide 17: HCP start/end, SG averages, biggest improvement, mental game pattern, practice structure split, programme adjustment.
STEP 4 — ADD PLAYBOOK LINKS
At the top of your journal page, add bookmarks to your most-used playbooks. Type /bookmark → paste a playbook URL. Add Guide 17, Guide 26, and Guide 10 at minimum. This makes the journal a true hub for the programme.
STEP 5 — COPY THE URL INTO THIS APP
In Notion, click Share on your journal page → copy the link. In this tracker → Dashboard → Settings → Notion Journal URL → paste → Save. The Journal tab button will then open directly to your Notion workspace.

📱 Step 3 — Install on Your Devices
Add to Home Screen — Both Phone and Laptop

One-Tap Access on Every Device

iPhone · Safari
Share button → "Add to Home Screen" → Add. Always use Safari — Chrome on iPhone has separate localStorage. Sign in once; Firebase keeps you signed in permanently.
Android · Chrome
⋮ menu → "Add to Home Screen" → Add. Sign in once; stays signed in.
Laptop · Any Browser
Open the GitHub Pages URL → bookmark it or pin the tab. Sign in once. Same Firebase account = same data as your phone, automatically in sync.
